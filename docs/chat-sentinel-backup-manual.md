# SillyTavern 聊天记录守护备份说明书

## 这是什么

`聊天记录守护备份` 是一个本地 SillyTavern 备份插件。

它的作用不是替代 SillyTavern 原本的聊天保存，而是在原本保存之外，再额外把当前聊天写成一份独立的 `.jsonl` 快照文件。这样如果聊天列表异常、浏览器缓存出问题、原插件 IndexedDB 被清掉、或者内置备份因为文件名冲突被覆盖，你还有另一层本地文件级备份可以翻。

它不会调用 OpenAI、Claude 或任何外部模型 API，也不会上传聊天内容。它只调用你本机 SillyTavern 的本地接口。

## 它解决什么问题

它主要防这几类问题：

- 聊天列表里某个会话突然不见了。
- SillyTavern 当前聊天文件被覆盖后，内置备份也没留下你想要的版本。
- 中文角色名在内置备份里被压成类似 `chat____...jsonl`，不同中文角色可能共用相似备份前缀，导致旧备份被清理。
- 浏览器 IndexedDB 备份插件的数据因为浏览器站点数据、缓存、配置迁移等原因不好找或丢失。

它不解决这几类问题：

- 不能凭空恢复插件安装之前已经消失的聊天。
- 不能恢复从来没有在 SillyTavern 前端加载过的内容。
- 如果 SillyTavern 没启动、服务器插件没加载、或者磁盘不可写，它不能写新快照。

## 怎么启用

1. 重启 SillyTavern。

   服务器插件只会在 SillyTavern 启动时加载。只刷新浏览器页面不够。

2. 打开 SillyTavern 页面。

3. 进入扩展设置区域，找到：

   `聊天记录守护备份`

4. 确认 `启用守护备份` 是勾选状态。

默认是启用的。

## 页面里每个选项是什么意思

`启用守护备份`

开启后，聊天发生变化时会自动排队写入快照。关闭后，自动快照暂停。

`间隔秒数`

同一个聊天自动备份的最短间隔。默认是 `20` 秒，允许范围是 `5` 到 `300` 秒。

它的意思是：如果你连续发消息、删消息、切 swipe，插件不会每一下都立刻写盘，而是合并成较少的快照，避免太频繁写文件。

`每聊保留`

每一个聊天最多保留多少份守护快照。默认是 `80` 份，允许范围是 `5` 到 `500` 份。

这里的“每聊”是按角色或群聊加聊天 ID 区分，不是所有角色共用一个总数。

`立即备份`

手动对当前打开的聊天写一份快照。你刚写完重要剧情、准备切聊天、准备重启、或者感觉列表不稳定时，可以点这个。

`备份本卡全部聊天`

把当前角色卡名下的所有聊天记录和分支，一次性写入守护快照。

如果当前打开的是群聊，它会备份当前群聊名下登记的所有群聊聊天文件。

这个按钮适合第一次安装插件后使用。比如 `星球` 下面已经有很多历史聊天和 `Branch #1`、`Branch #2` 之类的分支，你不需要逐个打开再点 `立即备份`，可以直接点一次 `备份本卡全部聊天`。

`选择聊天备份`

展开当前角色卡或群聊下的聊天文件列表，让你自己勾选要备份的聊天。

列表里会显示聊天文件名、消息数、文件大小和最近修改时间。勾选后点 `备份已选`，只会备份你选中的那些聊天。这个适合一个角色卡下面聊天很多，但你只想保护其中几条主线或分支的时候使用。

`管理当前版本`

展开当前打开这条聊天的所有守护快照版本，让你选择哪些版本要长期保留。

同一条聊天会因为发送、删除、修改、切 swipe、生成结束等动作产生很多快照。你可以在这里勾选某几个版本，点 `保留已选版本`。被保留的版本会在文件名里加上 `KEEP`，之后自动清理旧快照时不会删它。

如果后面不想特殊保留某个版本，可以勾选它再点 `取消保留`。取消保留不会马上删除文件，只是让它重新回到普通快照，之后才会按 `每聊保留` 的规则参与清理。

`刷新状态`

重新读取最近的守护快照列表，并显示快照目录。

## 它什么时候会自动备份

插件会在这些场景触发自动快照：

- 发出消息
- 收到回复
- 编辑消息
- 删除消息
- 删除多条消息
- 切换 swipe
- 一轮生成结束
- 切换聊天
- 选择角色首条消息
- 编辑或删除 reasoning
- 嵌入消息文件
- 页面初始化后
- 页面关闭前如果还有待写入快照，会尝试补一次

自动触发后，插件会先按设置尝试执行 SillyTavern 原生保存，然后再写守护快照。

自动备份只守护“当前打开的聊天”。如果一个角色卡下面有很多旧聊天，插件不会在后台偷偷扫完整个角色卡目录。要备份这个角色卡全部历史聊天，需要手动点 `备份本卡全部聊天`。

## 有分支怎么办

分支在 SillyTavern 里通常也是单独的 `.jsonl` 聊天文件，例如：

`星球 - 2026-06-26@20h37m07s665ms.jsonl`

`星球 - 2026-06-26@20h37m07s665ms - Branch #1.jsonl`

插件会把它们当成不同聊天文件处理。

如果你只点 `立即备份`，它只备份当前打开的那一条主线或分支。

如果你点 `备份本卡全部聊天`，它会把当前角色卡目录下所有 `.jsonl` 都备份进去，包括主线和分支。

## 一个角色卡有很多聊天记录怎么办

一个角色卡下面可以有很多聊天记录。插件不会只按角色名粗暴混成一份，而是按具体聊天文件分别保存。

例如 `星球` 下面有 20 个聊天文件：

- 自动备份：只备份你当前打开、发生变化的那一个。
- `立即备份`：只备份你当前打开的那一个。
- `备份本卡全部聊天`：备份 `星球` 目录下所有 `.jsonl`。
- `选择聊天备份`：列出 `星球` 目录下所有 `.jsonl`，你勾哪个就备份哪个。
- `管理当前版本`：不是选择聊天文件，而是选择当前这条聊天已经生成的多个快照版本，决定哪些版本要加 `KEEP` 长期保留。

每个聊天文件都会有自己的守护快照组，文件名里也会带原聊天名、哈希和消息数，所以不同聊天、不同分支不会互相覆盖。

## 一个聊天有很多快照版本怎么办

这是正常的。你删除、修改、继续生成、切换 swipe，都可能让同一条聊天产生新的快照。

如果某个版本特别重要，比如删改前的剧情、某次生成前的状态、或者你怕后面被覆盖的节点：

1. 打开那条聊天。
2. 点 `管理当前版本`。
3. 在快照列表里按时间、消息数、文件大小判断要留哪几个。
4. 勾选它们。
5. 点 `保留已选版本`。

被保留的快照文件名会带 `KEEP`，自动清理不会删它们。

这个功能不会自动判断哪个剧情最好，也不会删除没选中的版本。它只是把你选中的版本标记为长期保留，避免误清理。

## 它会写出什么文件

快照是 `.jsonl` 文件，保存在：

`data/default-user/backups/sentinel-chat`

文件名大概长这样：

`20260627-123456_char_星球_xxx_1234abcd5678ef90_m166.jsonl`

其中：

- `20260627-123456` 是写入时间。
- `char` 表示角色聊天，`group` 表示群聊。
- `星球_xxx` 是角色名和聊天 ID 的清理版。
- `1234abcd5678ef90` 是区分具体聊天的哈希。
- `m166` 表示这份快照里有 166 条消息。

这个命名方式故意保留可读名字，并额外加哈希，避免中文名都变成 `____` 后互相覆盖。

## 怎么确认它正在工作

最简单的方法：

1. 重启 SillyTavern。
2. 打开任意一个已有聊天。
3. 在 `聊天记录守护备份` 里点 `立即备份`。
4. 看状态栏是否出现类似：

   `已备份 166 条消息：20260627-xxxxxx_char_...jsonl`

5. 点 `刷新状态`，看列表里是否出现新文件。

也可以直接去这个目录看文件是否变多：

`/Users/money/SillyTavern-Launcher/SillyTavern/data/default-user/backups/sentinel-chat`

## 出事时怎么恢复

如果以后某个聊天又突然消失，先不要反复切换、删除、清理缓存。按这个顺序做：

1. 进入快照目录：

   `/Users/money/SillyTavern-Launcher/SillyTavern/data/default-user/backups/sentinel-chat`

2. 找对应角色名、聊天 ID、消息数的快照。

   例如文件名里有 `星球`、`m217`，就很可能是 `星球` 的 217 条版本。

3. 把这份 `.jsonl` 复制到对应角色聊天目录。

   例如 `星球` 的聊天目录是：

   `/Users/money/SillyTavern-Launcher/SillyTavern/data/default-user/chats/星球`

4. 给复制过去的文件起一个不冲突的名字，例如：

   `星球 - recovered-20260627-123456.jsonl`

5. 回到 SillyTavern，刷新聊天列表，选择这个 recovered 聊天。

建议恢复时先复制，不要移动原快照。快照目录里的文件保留着更安全。

## 和 SillyTavern 原生备份的区别

SillyTavern 原生备份也会保存聊天，但它有几个风险：

- 中文角色名可能被清理成很多下划线。
- 多个中文名可能落到相似文件名前缀。
- 原生备份按规则清理旧备份，旧版本可能被挤掉。

守护备份的区别是：

- 单独保存到 `sentinel-chat` 目录。
- 文件名带角色名和聊天哈希。
- 每个聊天单独保留一定数量。
- 拒绝写入空聊天，避免空内容覆盖有用快照。
- 如果内容完全重复，会跳过重复快照。

## 和旧聊天备份扩展的区别

旧扩展主要把备份存在浏览器 IndexedDB 里。

守护备份把内容写到 SillyTavern 用户目录下的真实 `.jsonl` 文件里。

所以它更容易直接查找、复制、恢复，也更不依赖浏览器缓存状态。

## 这次 217 条记录能靠它恢复吗

不能保证。

这个插件是在这次问题之后加上的。它能保护以后新产生、被它实际快照过的聊天；但对于它安装前已经消失、且没有留在当前聊天文件、内置备份、废纸篓或旧 IndexedDB 备份里的版本，它不能凭空生成。

目前查到的 `星球` 可恢复版本里，最接近的是 166 条，不是你说的 217 条。

如果你记得那段 217 条聊天里任意一句很独特的原文，可以继续用那句话去全盘搜索，包括旧浏览器备份 blob，仍然有一点机会。

## 日常建议

- 写到很重要的剧情节点时，手动点一次 `立即备份`。
- 不要把 `每聊保留` 设太低，默认 `80` 比较稳。
- 如果聊天很长、很珍贵，可以设到 `150` 或 `200`。
- 不要把 `sentinel-chat` 目录当缓存清理。
- 更新、迁移、重装 SillyTavern 前，把 `data/default-user/backups/sentinel-chat` 一起备份。

## 快速版

以后你只需要记住三件事：

1. 它会把当前聊天额外保存到本地文件：

   `/Users/money/SillyTavern-Launcher/SillyTavern/data/default-user/backups/sentinel-chat`

2. 重要剧情点可以点：

   `聊天记录守护备份` -> `立即备份`

3. 第一次安装后，或者一个角色卡下面聊天很多，可以点：

   `聊天记录守护备份` -> `备份本卡全部聊天`

4. 只想备份其中几条，可以点：

   `聊天记录守护备份` -> `选择聊天备份` -> 勾选 -> `备份已选`

5. 想保留当前聊天的某几个历史版本，可以点：

   `聊天记录守护备份` -> `管理当前版本` -> 勾选 -> `保留已选版本`

6. 真出事时，去快照目录找对应角色名、消息数和 `KEEP`，比如 `星球`、`m217`、`KEEP`，复制回角色聊天目录恢复。
